How The HTML Works

The four modes within the map list generator are all in a grid. This grid has the following CSS:

display: grid;
grid-template-columns: repeat(auto-fit, minmax(200px, 1fr));

which allows the columns stack on top of each other in rows when the screen size is compressed. Within each column, each map coresponds to a specific checkbox. Each checkbox is formatted as follows:

<label class="box" for="humpback-sz">
<input type="checkbox" name="humpback-sz" id="humpback-sz">
Humpback Pump Track
<span class="checkmark"></span><br>


The "box" and "checkmark" classes allow for the checkbox and checkmark to be styled in CSS. The id of each checkbox is used by the Javascript to know which maps are selected by the user. They are always formatted as the shorthand name of the map followed by the two letter abreviation of the name separated by a dash (ie. humpback-sz). The five different buttons are all in a grid in a similar format to the grid of the map names. The "Generate" button has the format:

<button type="button" class="button" onclick="makeMapList()">Generate</button>

The buttons are much easier to style than the checkboxes and radio buttons as they can easily be accessed by the class "button". The Javascript functions are accessed by the "onclick" element which calls the specific function when the user clicks the button. The radio buttons have a similar format to the checkboxes due to their difficult nature to style:

<label class="radio" for="sz-only">
<input type="radio" name="options" id="sz-only" onclick="burtOnly(); szOnly();">
SZ Only
<span class="radio-button"></span>
</label>

The classes "radio" and "radio-button" are used to style the radio buttons. The id is used by the Javascript to access the state of the button. "onclick" calls multiple Javascript functions which will be explained in the Javascript section. The area for the map list to be outputted and the "copy" button are both in a grid with two rows to ensure the copy button is always beneath the output area. The output area is a "p" element with the id "maplist" which allows for the Javascript to output to that specific element. The "copy" buttton is formatted similarily to the previous two buttons.

How The Javascript Works

Within the Javascript folder there are 5 main functions, each with their own file. The functions are makeMapList(), clearMaps(), szOnly(), burtOnly(), and copyText().

makeMapList is the most important function as it reads the states of each of the checkboxes and generates a random map list from the selected maps. Additionally, the maps will have a random order and follow this order until there are no more maps in a specific mode. For example if I have 3 zones maps, 2 tower, 0 rainmaker, and 2 clams, the function will generate a random order for the modes, say CB > SZ > RM > TC, so the output will look like:

CB, SZ, TC, CB, SZ, TC, SZ

As someone who frequently hosts lobbies, I find it valuable to cycle through all of the maps in a map list before repeating maps previously played just to keep a specific mode order in place.

The function starts by defining an array containing strings with all the different shorthand names for the maps, and array containing strings with all the official names of the maps, and an array containing the two letter capitalized abbreviations of the modes. These are used to avoid using multiple "if else" statements for each different map, instead these arrays can be referenced to get the desired strings. For example, the array containing all the shorthand names for the maps looks like this:

let mapArr = ["humpback", "inkblot", "moray", "mussel", "port", "starfish", "sturgeon", "reef", "manta", "kelp", "snapper", "blackbelly", "mako", "walleye", "dorf", "mall", "joe", "pit", "camp", "wahoo", "hotel", "ancho", "skipper"];

Four arrays are then declared for each of the modes and they are set to the values returned by four different functions, each function being titled read "Mode" Maps. These arrays will contain all the proper names for the maps selected by the user, ie SZ Inkblot Art Academy instead of the shorthand inkblot-sz used in the checkbox id's.

Within each reading function, a blank array is declared. This is where the output maps will be pushed to. The function then itterates through each of the values of the mapArr. A string is then made with the value of mapArr at i, mapArr[i] added with a dash followed by the lowercase shorthand for the modes. These created strings directly correlate to the id's of the checkboxes in the html. A variable is then set to the checkbox object of the specific map-mode combination. If the checkbox's state is checked, the map is pushed onto the blank array. This is repeated for all maps for a given mode, resulting in all the selected maps for a given mode being added to the specific mode array:

function readZonesMaps(mapArr, finalMapArr) {

let szArr = [];

for (let i = 0; i < mapArr.length; i++) {

let str = mapArr[i] + "-sz";
let checkbox = document.getElementById(str);

if (checkbox.checked == true) {

szArr.push("SZ " + finalMapArr[i]);

}

}

return szArr;

}

The only difference in the reading functions is that the strings for the modes are changed to match the mode being read.

Now that all the selected maps are sorted by mode, the random mode order is generated:

function getRandomModeOrder(modeArr) {

let modeOrderArr = [];

do {

let randomModeIndex = Math.floor(getRandomFactor() * modeArr.length);
modeOrderArr.push(modeArr[randomModeIndex]);
modeArr.splice(randomModeIndex, 1);

}

while(modeOrderArr.length < 4);

return modeOrderArr;

}

This code will assign a random number between 0 and 3 to the randomModeIndex variable. This mode will then be pushed to the modeOrderArr array and spliced out from the original modeArr array. The end result is an array of length four with the modes in a random order.

Now an array will be made with all the selected maps in the random mode order previously generated:

function generateMapListArr(szArr, tcArr, rmArr, cbArr, modeOrderArr) {

let totalMaps = szArr.length + tcArr.length + rmArr.length + cbArr.length;
let finalMapArr = [];
let j = 0;

for (let i = 0; i < totalMaps; i++) {

let mode = j % 4;

if(modeOrderArr[mode] == "SZ" && szArr.length > 0) {

let randomMapIndex = Math.floor(getRandomFactor() * szArr.length);

finalMapArr.push(szArr[randomMapIndex]);
szArr.splice(randomMapIndex, 1);

i++;

}

else if(modeOrderArr[mode] == "TC" && tcArr.length > 0) {

let randomMapIndex = Math.floor(getRandomFactor() * tcArr.length);

finalMapArr.push(tcArr[randomMapIndex]);
tcArr.splice(randomMapIndex, 1);

i++;

}

else if(modeOrderArr[mode] == "RM" && rmArr.length > 0) {

let randomMapIndex = Math.floor(getRandomFactor() * rmArr.length);

finalMapArr.push(rmArr[randomMapIndex]);
rmArr.splice(randomMapIndex, 1);

i++;

}

else if(modeOrderArr[mode] == "CB" && cbArr.length > 0) {

let randomMapIndex = Math.floor(getRandomFactor() * cbArr.length);

finalMapArr.push(cbArr[randomMapIndex]);
cbArr.splice(randomMapIndex, 1);

i++;

}

i--;
j++;

}

return finalMapArr;

}

 There is a specific if statement for each mode, to deal with the differing strings. However, these if statments all have the same function. First, the mode is determined by taking the modulus of the order of the map currently being added (for example if the map being added is map 17, the mode will be the value at the index 17 % 4 of the modeOrderArr). Once the mode is determined, the corresponding if statement is then read. The if statment will only go through if the mode matches and if there are a non-zero amount of maps in the array of maps for that mode. If both are true, a map is randomly picked from the maps within the specific mode array. This specific map is then pushed onto a final map list array and spliced off the original array. "i" is incremented again upon leaving the if statments as it is decremented at the end of the for loop. This is done to ensure that all the maps selected are pushed onto the final map list array. Say for example we get to map 16, i = 15, and there are no clam maps in the cbArr. Since cbArr.length is not greater than zero, by the time i exits the for loop it is still 15, since it is decremented at the end and then incremented by the for loop itself. This means that the function will go on to fill map 16 with a map from a mode that has a non-zero amount of maps. "j" is incremented at the end of the code for the same reason. If there are a non-zero amount of maps for a certain mode, we would want to move to the next mode instead of infinitely looping at a mode with no maps. So, regardless of if a map is added to the final array or not, the mode will always follow the order given by modeOrderArr.

Now the generateMapListStr function is used to generate the string that will be outputted back to the HTML.

function generateMapListStr(finalMapListArr) {

let finalMapListStr = "";

for (let i = 0; i < finalMapListArr.length; i++) {

let strToAdd = "";
let mapNum = i + 1;

if (mapNum == finalMapListArr.length) {

let strToAdd = mapNum + ") " + finalMapListArr[i];
finalMapListStr = finalMapListStr.concat(strToAdd);
}

else {

let strToAdd = mapNum + ") " + finalMapListArr[i] + "<br>";
finalMapListStr = finalMapListStr.concat(strToAdd);

}

}

return finalMapListStr;

}

This function takes each element of the finalMapListArr and adds them to a string. If the specific map is not the final map, the string will have a "<br>" so that in the HTML each map is separated by a break. The final string is then returned back to the main function.

let mapListStr = generateMapListStr(finalMapListArr);

document.getElementById("maplist").innerHTML = mapListStr;

The previously mentioned "p" element in the HTML where the map list will be outputted is then accessed by its id, "maplist", where it is set to the value of our final string.

The other Javascript used involves the five other buttons, "Clear", "SZ Only", "Burt's Extravaganza 4.0", "Custom", and "Copy".

The "Clear" button calls the function clearMaps(). This function will use a similar method used in the makeMapList() function to itterate through each map-mode combination. The function will check the state of the checkbox and if it is checked it will set it to be unchecked.

The "SZ Only" button calls the function szOnly(). SZ only is a fairly simple function, it access the column that the modes tower, rainmaker, and clam blitz are located in and sets their display to "none". In order to make sure that this only happens when the "SZ Only" radio button is selected, the function checks to make sure that the radio button is checked and will only change the display to "none" if it is checked. If it is not checked, the display will be set to "block". This is why "SZ Only" is called for the other radio buttons as it ensures that the tower, rainmaker, and clam blitz columns are not hidden. Additionally, the function will also run the code found in the clearMaps() function that affects the tower, rainmaker, and clam blitz checkboxes to ensure that the map list will only receive inputs from the visible splat zones column.

The "Burt's Extravaganza 4.0" button calls the burtOnly() function. This function does a couple of things. First, it checks to make sure that the respective button is checked. If it is not, the "disabled" state of each of the checkboxes is set to false. This allows for users to interact with the checkboxes; when the button is pressed we want the users to not be able to change the maps as it is a predetermined map list. If the user wants, they are able to hit the "Burt's Extravaganza 4.0" then hit the "custom" button to change the maps. However, if the "Burt's Extravaganza 4.0" is checked, the function will itterate through each possible map-mode combination and only change its state when the specific map is found in the array containing all the "Burt" maps. This is done with a nested for loop. The first for loop will itterate through all the possible map-mode combinations while the inner loop will itterate through all the "Burt" maps. If the specific map-mode combination has the same value as one of the maps in the "Burt" maps array, a count is itterated. Leaving the inner for loop, if the count is non-zero then the state of the checkbox is changed to "checked" and the "disabled" state is set to true. If the count is zero, the "disabled" state is still set to true but the checkbox "check" state is set to false. This results in all the "Burt" maps being selected in the checkboxes while all of the checkboxes cannot have their state change while the radio-button is checked. This function is called for the other two radio buttons to ensure that they do not keep the "disabled" state as true as this would not allow users to change the maps for other radio button options.

The "custom" button calls both the szOnly(); function and the burtOnly(); function for previously stated reasons: this ensures that the user has the most freedom while picking their maps for this option.

Last is the "copy" button. This button calls the function copyText(). This function copies the text with the id "maplist", which is the outputted maplist from the makeMapList() function, to the clipboard. A small adjustment was made as the method used to copy the text will force the user to highlight the text. In order to avoid an eyesore, the highlight status of the text with the id "maplist" was given an alpha value of 0, so when the user attempts to highlight the text it does not visibly look highlighted.
